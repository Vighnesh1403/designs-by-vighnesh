Event Booking System – End-to-End Experience Design

The project aimed to simplify the event booking and management flow for both admins and clients of a photography company. Previously, when users booked a multi-event package (like a wedding with multiple ceremonies), the backend system created each sub-event as a separate booking, causing confusion and inefficiency for both the operations and the clients.

I redesigned the entire system — from quotation creation to event tracking and post-production coordination — ensuring clarity, automation, and seamless communication between users and the admin team.

UX ResearchUser Flow DesignMobile App

Project Overview

  • Problem Statement

    Managing multi-event wedding bookings created multiple separate records for a single client, leading to confusion, inefficient coordination, and a poor client experience.

  • Solution

    Redesigned the entire platform with an integrated Client App + Admin Dashboard, allowing users to book multiple sub-events under a single quotation, with synchronized workflows for quotation creation, booking confirmation, event coordination, and post-production management.

  • Impact

    Simplified the booking workflow, reduced admin workload by over 60%, and improved client satisfaction through seamless communication, structured event tracking, and transparent process visibility.

Responsibilities

  • User Flow Analysis

    Researched and mapped the existing booking process to identify gaps, redundancies, and usability issues across multiple event workflows.

  • UX Redesign

    Redesigned the entire experience to unify multi-event bookings under a single flow, ensuring clarity and ease of use for both clients and admins.

  • UI Design

    Crafted clean, intuitive interfaces for the Client App and Admin Dashboard, focusing on hierarchy, readability, and consistency.

  • Collaboration & Handoff

    Worked closely with the development team to communicate design intent, provide specifications, and ensure smooth implementation.

1. Quotation Request & Creation Flow

Client View – Request Quote : The client begins by exploring various photography packages available on the platform. After selecting their preferred options, they fill out a short “Get Quote” form mentioning event type, date, and requirements. This request is then sent to the admin for review.

Admin View – Create Quotation : Once the request is received, the admin accesses the quotation dashboard, reviews client details, and generates a customized quotation. The quotation includes event details, pricing, and terms, which is then shared back with the client for approval.

Client View – Request Quote

The user begins by submitting a quotation request for their wedding or event package, specifying event types like Mehendi, Sangeet, Haldi, and Wedding.

Admin View – Create Quotation

The admin reviews the request, discusses package details with the client, and prepares a customized quotation that includes all sub-events, services, and costs.

2. Booking Confirmation & Coordination Form Fill-up Flow

Client Confirms Booking : Once the quotation is approved, the client proceeds to confirm the booking by completing payment or signing the service agreement. The system automatically updates the booking status and notifies the admin team for scheduling.

Coordination Form Fill-up : After confirmation, the client receives communication from the coordination team to finalize event details such as timing, location, and specific photography requirements. This ensures smooth execution on the event day and personalized service delivery.

Client View – Client Confirms Booking

Once the quotation is approved, the client pays 50% of the total amount to confirm the booking. The system automatically updates the booking status in both admin and client dashboards.

Client View – Coordination Form Fill-up

The client is prompted to fill in event-specific details like venue, timing, and contact information to help the photography team plan and coordinate smoothly.
